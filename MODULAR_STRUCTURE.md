# Модульная структура уникализации

## Новые модули

Создано 8 новых модульных уникализаторов для максимальной уникальности:

### 1. **BitDepthUniqueizer** (`bit_depth.py`)
- **Параметр:** Битовая глубина PNG
- **Значения:** 8, 16 бит
- **Что делает:** Изменяет битовую глубину PNG изображения

### 2. **ColorTypeUniqueizer** (`color_type.py`)
- **Параметр:** Цветовой тип PNG
- **Значения:** RGB, RGBA
- **Что делает:** Изменяет цветовой тип (RGB/RGBA)

### 3. **PNGFilterUniqueizer** (`png_filter.py`)
- **Параметр:** PNG фильтр
- **Значения:** Adaptive, None, Sub, Up, Average, Paeth
- **Что делает:** Изменяет метод фильтрации PNG

### 4. **InterlaceUniqueizer** (`interlace.py`)
- **Параметр:** Чересстрочная развёртка
- **Значения:** None (обычная), Progressive (прогрессивная)
- **Что делает:** Изменяет метод чересстрочной развёртки

### 5. **WhiteBalanceUniqueizer** (`white_balance.py`)
- **Параметр:** Баланс белого (EXIF)
- **Значения:** 0 (Auto), 1 (Manual)
- **Что делает:** Изменяет баланс белого в EXIF метаданных

### 6. **ApertureUniqueizer** (`aperture.py`)
- **Параметр:** Диафрагма / F-число (EXIF)
- **Значения:** f/1.8, f/2.8, f/4.0, f/5.6, f/8.0, f/11, f/16
- **Что делает:** Изменяет значение диафрагмы в EXIF

### 7. **ResolutionUniqueizer** (`resolution.py`)
- **Параметр:** Разрешение изображения (DPI)
- **Значения:** 72, 96, 150, 200, 300 DPI
- **Что делает:** Изменяет разрешение изображения в EXIF

### 8. **CompressionUniqueizer** (`compression.py`)
- **Параметр:** Тип сжатия PNG
- **Значения:** Deflate/Inflate (уровни 0-9)
- **Что делает:** Изменяет уровень сжатия PNG

---

## Интеграция

### В `AllCombinedUniqueizer`:
- Все 8 модулей добавлены в `__init__`
- Применяются случайным образом (3-5 модулей из 8) для каждой копии
- Это обеспечивает максимальную уникальность каждой копии

### В `AllCombinedWithPixelUniqueizer`:
- Использует `AllCombinedUniqueizer`, который уже включает все модули
- Затем применяет pixel pattern overlay
- Каждая копия получает уникальную комбинацию модулей + pixel pattern

## Как работает

1. **При генерации копии:**
   - `all_combined.process()` применяет стандартные методы
   - Затем случайно выбирает 3-5 модулей из 8 новых
   - Каждый модуль изменяет свой параметр
   - Результат = уникальная комбинация всех изменений

2. **Для PIXEL метода:**
   - То же самое + pixel pattern overlay
   - Каждая копия = уникальная комбинация модулей + уникальный pixel pattern

## Преимущества модульной структуры

✅ **Понятно:** Каждый модуль отвечает за один параметр
✅ **Модульно:** Легко добавлять/удалять модули
✅ **Гибко:** Можно применять разные комбинации
✅ **Уникально:** Каждая копия получает случайную комбинацию модулей
